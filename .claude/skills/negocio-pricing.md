# Modelo de Pricing Tulkas

## Principios
- Precio justo en la media baja del mercado LATAM
- Caro solo si el producto no tiene competencia directa
- Trial máximo 15 días → decisión rápida
- Sin freemium eterno
- Siempre 3 planes: básico / profesional / enterprise

## Estructura de 3 planes

### Plan Básico
- Lo mínimo para que un cliente pequeño empiece
- Sin todas las features avanzadas
- Precio: accesible para freelancers o PyMEs chicas
- Límites claros (usuarios, transacciones, volumen)

### Plan Profesional (el que más se vende)
- Features completas del producto
- Sin límites operativos restrictivos
- Precio: 2-3x el básico
- El reseller lo vende a sus clientes

### Plan Enterprise
- Sin límites + soporte prioritario + onboarding dedicado
- Precio: negociable, base 5x el básico
- Para empresas o municipios con múltiples usuarios

## Margen para reseller
El reseller tiene que poder hacer negocio:
- Precio Tulkas al reseller: $X
- Precio sugerido al cliente final: $X × 1.5 a 2
- Margen reseller: 50-100%
- El reseller no puede competir con Tulkas directo (precio diferenciado)

## Benchmarks por producto
- **OOH Planner**: agencias pagan $200-500/mes por herramientas similares
- **Tolrank**: agencias SEO pagan $100-300/mes por auditorías
- **Comunas**: municipios tienen presupuesto de $500-2000/mes para software
- **Plan-B**: negocios pagan $50-200/mes por WhatsApp Business API
- **Curex**: publishers pagan % de revenue o $200-500/mes

## Proceso para definir precio
1. ¿Cuánto paga el cliente por la solución actual?
2. ¿Cuánto ahorra/gana con nuestro producto?
3. Precio = 20-30% del valor generado
4. Verificar que sea < precio competidor con features similares
5. Calcular si el reseller puede hacer margen

## Monedas en LATAM
- Chile, México, Colombia, Perú: USD sin problema
- Argentina: USD preferido, ARS indexado como alternativa
- Nunca precio solo en ARS sin indexación
- MercadoPago para pagos locales, Stripe para internacional
